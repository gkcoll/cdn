What is GkCDN?

In Celestial Empire, there is an invisible and intangible wall in front of its netizens, it's the Great Firewall (GFW). In the past years, it has really done a good job in isolate and filter foreign Anti-Celestial-Empire speech.

But now, what the developers in Celestial Empire saw is it ban too many websites, there are many technical exchange websites among them. Although GitHub is not on the GFW blacklist, here are also difficult to visit it. It often inaccessible and unstable, even its raw domain name (raw.githuusercontent.com) cannot be accessed.

I originally thought that some local platforms within Celestial Empire could solve this problem, but they are all engaged in anti-stealing-link and content review. In order to finish the homework assigned by the Celestial Empire, they never pay attention to maintain an open-source environment at all.

In order to create a CDN service of my own, I found Vercel by Big data, which led to the project.

This project use free bandwidth provided by Vercel and its Global CDN acceleration, create a proxy service for the designated GitHub repository. Make developers who can't visit GitHub and GitHub raw data links legally can visit them in another way.

Start Building Your Own CDN Service

Before all operation, you should have a your own domain name. Like my gkcoll.xyz. The Vercel's vercel.app domain may unable to visit normally. We will resolve a subdomain (e.g cdn.gkcoll.xyz) to Vercel's server.

Register Vercel


Vercel is a easy way to create & deploy a online project. It's a cloud platform that supported import project from a GitHub repository, one click deployment, automatic update, everything in FREE (The Hobby Plan is enough for self using, and for more require you can try Pro Plan).

Vercel also supported bind custom domain name, it's useful when their domain names are banned in some district. In addition, Vercel will apply to Let's Encrypt for generate an SSL certificate for your domain name after you bound a domain name (within a few seconds), so you don't have to worry about the issue of browsers prompts that your website is not secure.

For register it, I suggest you choose authorize login with Github. But But it's not over yet, you still need to bind your phone number and email address.

New a Repository


You need a repository to store your static files. Open github.com, click on the left sidebar.

Complete your information in the opening page, here are some recommendation:

  • Repository Name: cdn
  • Visibility: Public
  • License: WTFPL (It's not on the preselection list, you can choose to manually create a LICENSE file and paste the license content later on)

Why we need it?

This project is mainly aimed at self-maintained projects, and these files generally need to be updated frequently, and even serve as configuration files. Originally, JsDelivr was the best solution, but now due to some irresistible causes, it cannot be accessed in China (otherwise it wouldn't be so troublesome). So, using this project and Vercel's global free bandwidth acceleration to access these files (it is not recommended to use it to store images here because it consumes too much space and bandwidth). For open-source projects with high popularity and high star on GitHub, my suggestion is not to manually proxy them yourself. Because there are better options -- static resource public library websites, such as JsDelivr, BootCDN, Staticfile CDN. They are all out-of-the-box and you needn't worry about the accessibility of them.
